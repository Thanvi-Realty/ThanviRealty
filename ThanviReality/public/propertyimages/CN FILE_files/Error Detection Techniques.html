  • Cannot detect two-bit or burst errors.

🔹 2. Checksum

A checksum is a value calculated from a block of data, used to verify its integrity after transmission.

  • Data is divided into equal-sized segments.

  • All segments are added using binary addition.

  • The 1’s complement of the sum is called the checksum.

  • The checksum is sent along with the data.

At the receiver end, the same operation is done. If the result is all 1s, data is accepted. Otherwise, error is detected.

✅ Example:

Data: 1010, 1100
Sum = 1010 + 1100 = 10110
Wrap around the carry: 0110
Checksum = 1’s complement of 0110 = 1001
Receiver checks: (data + checksum) should result in all 1s.

🔹 3. Cyclic Redundancy Check (CRC)

CRC is one of the most powerful error detection methods.

  • It treats the data as a large binary number.

  • Divides it by a predefined generator polynomial.

  • The remainder (CRC bits) is appended to the data and sent.

  • At the receiver side, the same division is done. If the remainder is zero, data is error-free.

✅ Example:

Data: 1101011011
Generator: 10011

  • Divide and get remainder (say: 0101)

  • Transmit: 11010110110101 (Data + CRC)

  • Receiver divides full code by 10011. If remainder ≠ 0 → Error detected.

✅ Advantages:

  • Detects burst errors better than parity and checksum.

  • Widely used in Ethernet, modems, disk drives.

🔹 4. Hamming Code

Hamming code is both an error detection and correction technique.

  • Adds redundant bits (called parity bits) at specific positions in data.

  • The position of these bits is powers of 2 (1, 2, 4, 8, etc.).

  • Helps in detecting and correcting single-bit errors.

✅ Example:

For 4-bit data 1011, we insert 3 parity bits → p1, p2, 1, p4, 0, 1, 1

  • Positions 1, 2, and 4 are for parity.

🔹 Cyclic Redundancy Check (CRC)

✅ Definition:

Cyclic Redundancy Check (CRC) is a powerful error detection technique used to detect accidental changes in data during transmission. It works based on binary division.

🔸 How CRC Works:

  1. Sender Side:

    • Data is treated as a binary number.

    • It is divided by a fixed binary divisor (called a generator polynomial).

    • The remainder from the division is called the CRC or checksum.

    • This CRC is appended to the original data and sent to the receiver.

  2. Receiver Side:

    • The received data (original data + CRC) is divided by the same generator.

    • If the remainder is 0, no error.

    • If remainder ≠ 0, error detected.

📌 Important Terms:

TermDescription
DataOriginal binary data to be sent
GeneratorPredefined binary pattern (like 1001, 1011, etc.)
CRCRemainder of division, added to data

✅ Example (Simple):

Let’s say:

  • Data = 1101

  • Generator = 1011 (4 bits → so append 3 zeros to data)

Step 1: Append 3 zeros to data → 1101000

Step 2: Divide 1101000 by 1011 using XOR (binary division)
➡ Remainder = 011

Step 3: Transmit 1101 + 011 → Final code sent = 1101011

Receiver Side:

  • Receives 1101011

  • Divides it by 1011

  • If remainder = 0 → No error

  • If remainder ≠ 0 → Error detected

✅ Advantages of CRC:

  • Detects single-bit errors, double-bit errors, and burst errors effectively.

  • More reliable than parity and checksum.

  • Widely used in network protocols, hard disks, modems, and USBs.

❌ Limitations of CRC:

  • Can detect errors but cannot correct them.

  • Slightly complex due to binary division logic.

  • Requires both sender and receiver to use same generator.

📚 Used In:

  • Ethernet frames (LAN)

  • HDLC (High-Level Data Link Control)

  • Storage devices (CDs, DVDs)

  • USB & communication protocols
