OSI Reference Model (Open Systems Interconnection Model)

The OSI model is a conceptual framework that standardizes the functions of a telecommunication or computing system into seven distinct layers. Each layer serves a specific function and communicates with the layers directly above and below it.

Diagram of OSI Model:

pgsql
+-------------------------+
| 7. Application Layer    | ← User interface (e.g., browsers, email)
+-------------------------+
| 6. Presentation Layer   | ← Data format, encryption, compression
+-------------------------+
| 5. Session Layer        | ← Connection management between applications
+-------------------------+
| 4. Transport Layer       | ← End-to-end communication, error recovery
+-------------------------+
| 3. Network Layer        | ← Routing, logical addressing (IP)
+-------------------------+
| 2. Data Link Layer      | ← MAC addressing, error detection
+-------------------------+
| 1. Physical Layer       | ← Transmission of raw bits (cables, signals)
+-------------------------+

Functions of Each Layer:

  1. Physical Layer

    • Deals with hardware transmission of raw bits over a physical medium.

    • Examples: Cables, switches, voltage levels.

  2. Data Link Layer

    • Ensures reliable transmission using MAC addresses and error detection.

    • Divided into MAC and LLC sublayers.

  3. Network Layer

    • Handles routing and logical addressing (IP addresses).

    • Example: IP, routers.

  4. Transport Layer

    • Provides reliable data transfer, flow control, and error handling.

    • Protocols: TCP, UDP.

  5. Session Layer

    • Manages sessions or connections between applications.

    • Responsible for dialog control and synchronization.

  6. Presentation Layer

    • Translates data formats (e.g., encryption, compression, encoding).

    • Ensures that data is readable by the application layer.

  7. Application Layer

    • Closest to the user. Provides network services to applications.

    • Examples: HTTP, FTP, SMTP, DNS.

Easy Mnemonic to Remember the Layers (Top to Bottom):

"All People Seem To Need Data Processing"

Comparison Between OSI and TCP/IP Models (Theory Only)

The OSI (Open Systems Interconnection) model and the TCP/IP (Transmission Control Protocol/Internet Protocol) model are two important reference models used in computer networks. Although both aim to facilitate communication between devices, they differ in structure, design, and application.

1. Number of Layers:

The OSI model has seven layers: Physical, Data Link, Network, Transport, Session, Presentation, and Application.
The TCP/IP model has four layers: Network Access, Internet, Transport, and Application.

2. Developed By:

The OSI model was developed by the International Organization for Standardization (ISO) in the late 1970s as a generic, protocol-independent framework.
The TCP/IP model was developed by the Department of Defense (DoD) in the 1970s to support the development of the internet.

3. Layer Functions:

In the OSI model, each layer has a distinct function. The Presentation and Session layers handle data format conversion and session control, respectively.
In contrast, the TCP/IP model combines multiple OSI layer functions into fewer layers. The Presentation and Session layer functionalities are included within the Application layer of the TCP/IP model.

4. Protocol Dependency:

The OSI model is protocol-independent and serves as a general framework for network communication. It does not specify specific protocols for each layer.
The TCP/IP model is protocol-specific and is based on standardized protocols such as TCP, IP, FTP, and HTTP.

5. Usage and Practicality:

The OSI model is mainly used as a teaching and reference model. It is not implemented directly in real-world communication systems.
The TCP/IP model is widely used in real-world networking, including the Internet. It is the foundation of modern internet protocols.

6. Top-Down vs. Bottom-Up Design:

The OSI model follows a top-down approach, where each layer was defined before the development of protocols.
The TCP/IP model follows a bottom-up approach, where the protocols were developed first, and then the model was designed around them.

7. Standardization and Flexibility:

The OSI model strictly defines the boundaries and roles of each layer, ensuring modular design.
The TCP/IP model is more flexible, allowing easier integration of protocols and more direct communication between layers.

Conclusion:

The OSI model provides a better theoretical understanding of network functions, while the TCP/IP model is more practical and forms the basis of today's internet communication. Despite differences, both models help in understanding and designing robust network architectures.

🔷 Circuit Switching and Packet Switching

These are two fundamental methods used for data transmission in computer networks and telecommunications.

✅ 1. Circuit Switching

Definition:
Circuit switching is a technique where a dedicated communication path is established between sender and receiver before the actual transmission begins. This path remains reserved for the entire session.

🔹 Diagram: Circuit Switching

css
[Sender]───╶╶╶ Dedicated Path ╶╶╶───[Receiver]

Key Features:

  • Dedicated path throughout the session

  • No sharing of the path with others

  • Fixed bandwidth

Example:
Traditional telephone networks use circuit switching. Once a call is made, a line is reserved until the call ends.

Advantages:

  • Reliable and constant connection

  • No delay once the circuit is established

Disadvantages:

  • Wastes resources if no data is being sent

  • Takes time to establish the circuit

✅ 2. Packet Switching

Definition:
In packet switching, data is divided into small units called packets, and each packet is routed independently through the network based on availability.

🔹 Diagram: Packet Switching

[Sender] → ▓→▓→▓  (Packets take different routes)  →▓→▓→▓ → [Receiver]

Key Features:

  • No dedicated path

  • Packets may arrive out of order

  • More efficient use of network

Example:
The Internet uses packet switching. Protocols like TCP/IP are based on this method.

Advantages:

  • Efficient use of bandwidth

  • Robust and fault-tolerant

Disadvantages:

  • Packets may get delayed or lost

  • Requires complex reassembly at the receiver end

✅ Differentiate Between Datagram and Virtual Circuit Networks

(Theory-Based Answer)

In packet-switched networks, data is transmitted in small units called packets. There are two main approaches to handle this: datagram networks and virtual circuit networks.

A datagram network is a connectionless packet-switching system. In this method, each packet is treated independently and is sent through the network without establishing a dedicated path. Since no connection is set up before data transfer, packets may take different routes to reach the destination. As a result, they may arrive out of order, get delayed, or even be lost during transmission. The responsibility of rearranging the packets lies with the receiver. Datagram networks are more flexible and efficient, especially in situations where short and irregular communication is required. An example of a datagram network is the Internet using IP (Internet Protocol).

In contrast, a virtual circuit network is a connection-oriented system. Before any actual data transfer happens, a logical path or “virtual circuit” is established between the sender and the receiver. All packets then follow this same pre-defined route, ensuring that they arrive in sequence and with minimal variation in delay. This approach is more reliable, as the path remains constant during the session and network resources can be managed more efficiently. However, setting up the virtual circuit takes additional time and overhead. Virtual circuit networks are suitable for real-time applications like voice and video calls, where order and timing are important. Examples include Frame Relay and Asynchronous Transfer Mode (ATM).

In summary, datagram networks offer more flexibility and speed without the need for setup, making them ideal for general internet traffic. Virtual circuit networks, on the other hand, provide a more reliable and ordered form of communication, making them suitable for time-sensitive services. While both methods are used in packet-switched communication, they differ significantly in connection setup, packet routing, reliability, and application areas.
